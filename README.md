# angel-fallen

Angel Fallen is a top-down 2D action roguelike built with Godot 4.

Current playable prototype includes:
- Main menu -> game scene transition
- Player movement, sprint, stamina
- Auto-attack combat against spawned enemies
- Enemy archetype system (normal/fast/tank/ranged) with chapter-based archetype mapping and elite wave injections
- Enemy object pooling (spawn/acquire/release/reset) to reduce allocation spikes in dense rooms
- Run-plan driven room generation (chapter-aware room metadata from `map_generation.json`)
- Treasure rooms generated by run-plan metadata (elite cache challenge -> loot payout, with per-chapter room-type caps for event/shop/elite/treasure)
- Room combat flow (clear room then press `E` for next)
- XP leveling with 3-choice level-up panel
- Shop room purchases and safe-camp forge/route choices
- Shop economy assist system (chapter-aware pricing profile, catch-up discount, high-gold markup, escalating restock, chapter-weighted quality/category bias)
- Boss progression now follows generated run-plan chapter metadata (default still 15-room MVP pacing)
- Boss phase behavior (dash + pulse) driven by HP thresholds
- Chapter-based environment hazards (water/spore/lava/ice/void) with status gauges
- Hazard visual feedback (chapter tint + damage flash) and boss-room hazard amplification
- Main gameplay scene visual baseline migrated to TileMap layers backed by resource tilesets (chapter ground + chapter-specific door block states)
- Main gameplay scene now includes a chapter-specific ground-detail TileMap layer for decorative sigils, cracks, frost marks, and void glyphs
- Hazard overlay visual layer migrated to resource-backed TileMap with chapter-specific overlay tilesets (hazard intensity drives overlay alpha)
- Hazard overlay now supports chapter-specific animation cadence (`hazard_anim_interval`) with atlas variant cycling
- Main gameplay scene now includes a chapter-specific ambient FX TileMap layer with atlas-cycled glow glyphs (`ambient_fx_interval` driven)
- Chapter visual profiles now drive detail tint/alpha pulse and ambient FX scroll/wave cadence (`visual_profile` in `environment_config.json`)
- Tile atlases are now generated in a handdrawn profile (`handdrawn_v1`) with an `atlas_manifest.json` snapshot for pipeline traceability
- Content-depth cycle D18 expanded roster/evolution/shop/memory coverage (characters 12, evolutions 13, shop passive 14, shop weapon 15, memory fragments 22)
- Resource directory families now use a mixed ready/candidate catalog baseline (`stub_catalog_v8`) with material-profile mapping plus preview/acceptance metadata, and batch-6 `production_ready` IDs validated via ratio + cross-scene smoke + scene-visual checks (including `bool_true`) with acceptance-profile tier semantics and layered trend baselines
- D36 visual snapshot regression chain now adds approval-threshold template orchestration and release-candidate tracking (`approval_threshold_templates` + `release_candidate_tracking`) on top of approval history archive aggregation (`approval_history_archive`), approval audit persistence (`approval_audit_trail`), backend matrix governance + approval workflow hard gates (`backend_matrix_governance` + `approval_workflow`), release template manifest governance (`release_gate_templates`), strategy orchestration (`strategy_orchestration`), exception lifecycle governance (`exception_lifecycle`), cross-backend regression attribution (`backend_attribution`), whitelist convergence automation (`whitelist_convergence`), required snapshot alignment, backend capture-mode governance, whitelist-governed diff tolerance, precision convergence, cross-version baseline drift controls, and layered report convergence gates
- D30-1 adds a fast JSON syntax gate (`scripts/tools/check_json_syntax.py`) for all balance/catalog/atlas JSON files, enabling fail-fast parse diagnostics before semantic validators and heavier rehearsals
- Meta progression save with run summary rewards
- Route-based victory ending unlock and achievement tracking
- Run result panel with ending/achievement unlock summary
- Main menu meta-upgrade shop (permanent run-start bonuses)
- Chapter transition narrative panel after boss rooms (route choice impacts alignment)
- Data-driven narrative content config (`data/balance/narrative_content.json`)
- Chapter intro panel now follows run-plan intro markers (default 1/5/9/13)
- Narrative event rooms (chapter event choices with resource/route consequences)
- Narrative choices can attach lasting chapter effects (2-3 rooms of buffs/curses)
- Event branch selection now uses weighted routing (alignment + unlocked endings)
- Memory fragments unlocked from chapter bosses and tracked in meta save
- Safe-camp Memory Altar panel supports in-run fragment review
- Blessing system now safely swaps (no unintended stacking from repeated choices)
- Run result panel includes a narrative-choice timeline summary
- Ending unlock now includes epilogue text (first unlock vs repeat cycle variants)
- Main menu memory archive can cycle unlocked fragment full text
- Main menu character selection (Knight/Mage/Rogue) with persisted last pick
- Each character has a unique starting trait (Bulwark / Arcane Overload / Shadowstep)
- Character unlock rules: Knight starts unlocked, Mage unlocks via Redeem ending, Rogue unlocks via `ach_slayer_120`
- Character weapon profile is data-driven (single/spread, projectile count, spread angle, random jitter, pierce hits, visual style)
- Weapon evolution option now appears in level-up panel when recipe conditions are met (weapon + passive level)
- Main menu Codex overlay tracks all categories with catalog progress (locked + unlocked + unlock hints)

## CI (GitHub Actions)

Workflow file: `.github/workflows/ci.yml`

CI jobs:
- `validate-config-and-resources`: run JSON syntax gate (`scripts/tools/check_json_syntax.py`) before `scripts/validate_configs.py` and `scripts/check_resources.py`
- `godot-headless-and-gut`: run `godot --import`, headless startup check, GUT tests (unit + integration), quality baseline snapshot script (`scripts/tools/run_quality_baseline.gd`), release-gate rehearsal (`scripts/tools/run_release_gate.gd`), compatibility rehearsal (`scripts/tools/run_compatibility_rehearsal.gd`), resource-acceptance rehearsal with scene-visual checks (`scripts/tools/run_resource_acceptance.gd`), resource-acceptance trend+visual gate (second run), visual snapshot regression rehearsal (`scripts/tools/run_visual_snapshot_regression.gd` with `VISUAL_SNAPSHOT_BACKEND_TAG` + `VISUAL_SNAPSHOT_BACKEND_MATRIX` + `VISUAL_SNAPSHOT_STRATEGY=ci_rehearsal` + `VISUAL_SNAPSHOT_RUN_MODE=rehearsal` + `VISUAL_PIPELINE_ID`), visual snapshot trend gate (second run, `VISUAL_SNAPSHOT_STRATEGY=ci_trend_gate` + `VISUAL_SNAPSHOT_RUN_MODE=trend_gate` + `VISUAL_PIPELINE_ID`), and release template dry-runs (`VISUAL_SNAPSHOT_STRATEGY=release_candidate/release_blocking` + `VISUAL_PIPELINE_ID`) on `ubuntu-latest` + `windows-latest` matrix

Trigger:
- push (all branches)
- pull_request

Controls:
- `WASD` / Arrows: Move
- `Shift`: Sprint
- `E`: Next room (non-combat rooms / cleared combat rooms)
- Shop room: `1/2/3/4` buy slot, `R` restock
- Shop room: `T` exchange ore -> gold (limited trades per shop visit)
- Shop economy now shifts by chapter (`chapter_1`~`chapter_4`) with per-chapter price/restock/exchange tuning, quality-rate multipliers, and category bias
- Elite rooms are generated by run-plan metadata; they use elite-only combat waves and higher pressure
- Treasure rooms now run an elite mini-wave challenge before loot deployment; clear objective then collect rewards and press `E` to continue
- Route branching: after cleared rooms with multiple exits, use `1/2` to select route then `E` to confirm next room
- Route style now locks per chapter on first branch choice (`vanguard` / `raider`) and biases event weights, shop economy, hazard intensity, gold/ore drop tendency, route-tint visual feedback, and boss parameter profile (HP/Damage/Speed/Scale/Color)
- Level-up panel may show `Evolution: ...` option; selecting it upgrades current weapon profile according to `data/balance/evolutions.json`
- Safe camp: `F` damage forge (3 ore), `G` speed forge (5 ore)
- Safe camp: `T` open Memory Altar (in-run fragment archive)
- Safe camp route: `Q` holy route, `V` void route
- Chapter transition panel: `1/2` choose route, `E` confirm continue
- Narrative event panel: `1/2` choose event branch, `E` / `Esc` continue
- Chapter intro panel: `E` / `Esc` begin chapter
- Boss rooms: defeat 1 boss target to unlock doors and progress chapter
- Accessory drops: auto-equip up to 2 slots on pickup (extra converts to gold)
- Environment gauges: Frostbite/Void Corruption shown on HUD (builds in hazard rooms, decays in shop/camp)
- HUD Chapter Effects row: shows active timed effects with icon and remaining rooms (`[+]` boon / `[-]` curse / `[~]` mixed)
- HUD Route Style row: shows current style, per-chapter style comparison (`CH1~CH4`), and latest style-lock event
- HUD toast notifications for key milestones (achievement, ending, memory fragment, accessory, codex unlock)
- HUD minimap now prefers run-plan graph coordinates (`map_x/map_y` + `map_bounds`) for branch-aware layout, with index-grid fallback when coordinates are unavailable; unexplored rooms use `??`
- Accessories now include environment resistance bonuses (frost/void mitigation)
- `Esc`: Open/close Pause Panel during run
- Pause panel: `1` Resume, `2` Retreat (settle run), `3` Main Menu
- Pause panel settings subpage: `4` open settings, sliders apply instantly and persist
- Settings include Master/BGM/SFX/Ambience volume, Screen Shake strength, UI scale (0.8x-1.5x)
- Settings panel supports `Reset Defaults`, and changed values are highlighted in gold
- Settings panel includes audio bus meters (Master/BGM/SFX/Ambience) and `Play Test SFX` button
- Settings panel now supports input remap (keyboard + gamepad): click `Key`/`Pad` to listen, press target input to bind, `Esc` cancels capture
- Input remap has independent `Reset Input Defaults`; while listening, pause/menu close hotkeys are suppressed to prevent accidental panel close
- Run result panel: `E` / `Esc` to return menu
- Main menu memory archive: click `Review Next Memory Fragment` to cycle unlocked fragments
- Main menu ending archive: click `Review Next Ending` to cycle unlocked endings (story + first/repeat epilogue), with faction theme colors (Redeem/Fall/Balance)
- Run result panel now includes chapter-effect timeline replay for this run (room, chapter, effect icon, duration) with filter cycling (`ALL/BOON/CURSE/MIXED`, click or key `1`), mode tinting, in-session filter memory, view switch (`FLAT/GROUPED`, click or key `2`), grouped chapter-anchor navigation (`Q/E` or Anchor button), and grouped scope toggle (`ALL CHAPTERS` vs `ANCHOR ONLY`, click or key `3`)
- Run result panel now includes route-style evolution summary (chapter comparison + lock trajectory by room/chapter)
- Main menu now has a Settings overlay using the same runtime settings data as pause panel
- Main menu character switch: `Prev/Next` buttons, selection applies to next run and is saved
- Character trait is shown on selection panel and applied at run start before meta upgrades
- Main menu Codex: open with `Codex` button; use `Prev/Next` (or keys `1/2`) to browse categories, `Entry Prev/Next` (or keys `3/4`) to inspect entries, `Source/Chapter` filter buttons (or keys `5/6`) to filter statistics, `E/Esc` to close; detail pane shows status/hint/source/time/run index
- Codex statistics panel shows per-category completion bars, chapter completion bars (Chapter 1~4 + Global), global completion bar, and most recent 5 unlock records filtered by source/chapter (chapter comes from real unlock context metadata; filters persist across sessions)

Chapter transition blessing effects:
- `holy_vow`: +armor, +frost resistance, immediate heal
- `void_oath`: +weapon damage, +crit chance, -armor tradeoff
- Blessings are mutually exclusive: selecting a new blessing replaces the previous one

Chapter event lasting effects:
- Some event choices add a timed chapter effect for next 2-3 rooms
- Effects can modify enemy spawn rate/HP/damage, hazard intensity, move speed, damage bonus, and temporary armor delta
- Effects are shown in both room status and HUD Chapter Effects row, with remaining room count (`(nR)`)

Meta progression:
- Run ends on death / victory / retreat, shows result panel, then returns to menu
- Main menu shows meta currency, run stats, and last run reward
- Main menu also shows achievement/ending unlock progress and meta upgrades
- Main menu now lists unlocked memory fragments
- Save file path: `user://meta_save.json`
