## 1. 设计总览

### 1.1 设计理念

游戏采用 **3大章节 × 4层楼 + 最终章** 的结构，共计 **13个可玩楼层 + 2个隐藏楼层**。
每个章节拥有独立的视觉主题、环境机制和敌人生态，确保玩家在推进过程中持续获得新鲜感。

### 1.2 核心原则

- **递进式复杂度**: 每章引入1-2个新环境机制，逐步叠加
- **主题一致性**: 同章节内的楼层共享视觉语言，但各有变体
- **机制教学**: 每个新机制首次出现时有安全的学习环境
- **风险回报**: 高难度路径提供更好的奖励

### 1.3 整体结构一览

```
第一章: 遗忘深渊 (F1-F4)     → 基础教学 + 核心循环建立
  F1 废弃地牢    F2 地下水道    F3 蘑菇洞窟    F4 矿脉深处(章Boss)

第二章: 灼热炼狱 (F5-F8)     → 环境危害 + 高压战斗
  F5 熔岩前厅    F6 锻造大厅    F7 火山裂隙    F8 炎魔祭坛(章Boss)

第三章: 永冻禁域 (F9-F12)    → 资源管理 + 策略深度
  F9 冰封遗迹    F10 极光冰原   F11 冰晶迷宫   F12 寒霜王座(章Boss)

最终章: 虚空之心 (F13)       → 终极挑战
  F13 虚空核心(最终Boss)

隐藏层: FS1 时间裂缝 / FS2 创世熔炉
```

## 2. 世界观与章节结构

### 2.1 主角背景 — 堕落天使·艾瑟瑞尔

**身份:** 曾经的天界第七序列守护天使，神圣军团"黎明之翼"副指挥官。

**堕落起因:**
千年前，天界与深渊的最终战争中，艾瑟瑞尔奉命守护连接天界与人间的"创世之柱"。深渊领主奥比利翁发动奇袭，以虚空之力腐蚀创世之柱。艾瑟瑞尔为阻止柱体崩塌，以自身神圣之力强行封印裂缝——但虚空的侵蚀同时渗入了她的灵魂。

天界审判庭判定她已被虚空污染，将她的六翼折断四翼，剥夺神圣权柄，放逐至地下遗迹最深处——遗忘深渊。

**现状:**
- 仅存的两翼残破不全，无法飞行（游戏中表现为闪避/冲刺）
- 神圣之力大幅衰退，需要通过战斗重新觉醒
- 体内残留的虚空之力与神圣之力不断冲突（对应升级时的"光明路线"与"堕落路线"选择）
- 记忆碎片散落在各层（收集品系统，拼凑完整过去）

**角色成长主题:**
```
F1-F4  (遗忘深渊): 觉醒 — 从废墟中站起，重拾战斗本能
F5-F8  (灼热炼狱): 锻炼 — 在烈火中重铸意志，直面过去的战友
F9-F12 (永冻禁域): 抉择 — 在冰封的真相前选择：救赎还是堕落
F13    (虚空之心): 终局 — 面对当年的宿敌，了结千年因果
```

**双路线系统预览:**
| 路线 | 触发条件 | 外观变化 | 能力倾向 |
|------|---------|---------|---------|
| 救赎之路 | 升级时多次选择神圣系被动 | 残翼逐渐修复，金色光环 | 治愈、护盾、神圣伤害 |
| 堕落之路 | 升级时多次选择虚空系被动 | 翅膀变为暗紫色，眼部虚空裂纹 | 虚空侵蚀、生命汲取、混沌伤害 |
| 平衡之路 | 两系被动数量接近 | 一翼金色一翼紫色 | 均衡成长，解锁专属融合技能 |

### 2.2 世界观设定

**遗迹的真相:**
地下遗迹并非天然形成——它是千年前那场大战的战场遗骸。创世之柱的碎片散落各层，扭曲了周围的空间与生态：
- **遗忘深渊**: 创世之柱的基座残骸，神圣力量衰退后被亡灵和低等生物占据
- **灼热炼狱**: 创世之柱核心能量泄漏，形成永不熄灭的地火，吸引了火焰生物筑巢
- **永冻禁域**: 天界审判庭施加的封印之力凝结为永恒寒冰，封存着战争时期的秘密
- **虚空之心**: 奥比利翁当年撕开的虚空裂缝核心，至今仍在缓慢扩张

**关键NPC:**
| NPC | 位置 | 身份 | 作用 |
|-----|------|------|------|
| 莫里昂 | 安全营地 | 堕落的天使铸造师 | 商店、武器强化、主线剧情 |
| 薇拉 | 安全营地 | 人类考古学者 | 记忆碎片解读、图鉴、支线任务 |
| 塞拉菲姆 | F4/F8/F12 Boss后 | 天界监察使（全息投影） | 审判对话、路线分歧触发 |
| ??? | 隐藏层 | 被封印的第二位堕落天使 | 隐藏剧情、真结局线索 |

### 2.3 章节过渡剧情系统

章节过渡发生在击败章Boss后、进入安全营地时。采用 **全屏叙事画面 + 文字演出** 的形式，玩家可随时按键跳过。

#### 技术实现

```gdscript
# 章节过渡场景结构
class_name ChapterTransition extends Control

@export var chapter_id: int
@export var narrative_data: NarrativeResource

# 演出组件
@onready var bg_art: TextureRect          # 全屏插画背景
@onready var character_portrait: TextureRect  # 角色立绘
@onready var text_label: RichTextLabel     # 叙事文字（支持BBCode）
@onready var choice_container: VBoxContainer  # 选择分支（如有）
@onready var anim_player: AnimationPlayer  # 转场动画
@onready var bgm_player: AudioStreamPlayer # 背景音乐

func play_narrative():
    for segment in narrative_data.segments:
        bg_art.texture = segment.background
        character_portrait.texture = segment.portrait
        anim_player.play(segment.transition_anim)  # "fade_in" / "slide" / "dissolve"
        await _typewriter_text(segment.text, segment.speed)
        if segment.has_choice:
            var choice = await _show_choices(segment.choices)
            EventBus.narrative_choice.emit(chapter_id, segment.id, choice)
        else:
            await _wait_for_input()
    
    # 过渡结束，加载安全营地
    SceneManager.transition_to("res://scenes/safe_camp.tscn")
```

```gdscript
# 叙事数据资源
class_name NarrativeResource extends Resource
# 注意: 完整 Resource 定义见 data-structures.md Section 1.8

@export var chapter_id: String
@export var trigger: String                 # "chapter_start" / "chapter_end" / "ending"
@export var segments: Array[NarrativeSegment] = []
@export var background: Texture2D           # 默认背景
@export var bgm: AudioStream               # 默认BGM

class_name NarrativeSegment extends Resource
# 注意: 完整 Resource 定义见 data-structures.md Section 1.8

@export var id: String
@export var type: String = "narration"     # "narration" / "dialogue" / "choice" / "cutscene"
@export var speaker: String = ""           # 说话者名称（narration时为空）
@export var text: String                   # BBCode格式文字
@export var mood: String = "neutral"       # 角色情绪（影响立绘表情）
@export var speed: float = 0.03            # 打字机速度（秒/字）
@export var background: Texture2D          # 背景插画
@export var portrait: Texture2D            # 角色立绘（可为null）
@export var transition_anim: String = "fade_in"
@export var has_choice: bool = false
@export var choices: Array[Dictionary] = []  # [{text, route_flag, alignment_delta}]
@export var bgm_override: AudioStream      # 可选BGM切换
```

### 2.4 章节转换机制

- 每章最后一层击败章Boss后，播放 **章节过渡剧情**（上述叙事内容）
- 剧情结束后进入 **安全营地**（非战斗区域）：
  - 永久商店·莫里昂（消耗Meta货币，第一章后解锁）
  - 记忆碎片解读·薇拉（第二章后解锁）

- 阶段3 (50%-25% HP): 时空乱流
  - 随机切换房间环境（熔岩/冰原/孢子）
  - 时间加速/减速交替
  - 分身攻击（3个分身同时行动）
- 阶段4 (25%-0% HP): 终焉
  - 全屏虚空侵蚀（持续掉血）
  - 所有技能同时释放
  - DPS检测：60秒内未击杀则全屏即死

## 6. Boss 设计

### 6.1 Boss 设计原则

- 每个Boss有 **3-4个阶段**，每阶段引入新机制
- Boss战斗融合所在章节的环境机制
- 提供明确的攻击预警（红圈/闪光/音效）
- 每个Boss有至少一个 **可打断技能**（奖励高手玩家）
- Boss掉落独特的 **Boss专属道具**（每个Boss 3-4种，随机掉1个）

> **饰品（Accessory）** 为独立装备类型，最多装备2个，不占用被动槽位。

### 8.3 难度调节参数

> **DDA（动态难度调节）为可选辅助系统**，默认关闭，可在设置菜单中开启。
> 开启后仅微调敌人强度和掉落率（±50%范围内），不影响楼层结构和Boss设计。
> 不适用于隐藏关卡（FS1/FS2）。

```gdscript
# 动态难度调节 (DDA) — 可选辅助系统
class_name DifficultyAdjuster

# 根据玩家表现动态微调
var performance_score: float = 1.0  # 0.5(困难) ~ 1.5(轻松)

func update_performance(player_data: Dictionary):
    var hp_ratio = player_data.current_hp / player_data.max_hp
    var clear_speed = player_data.room_clear_time / expected_clear_time
    var damage_taken_ratio = player_data.damage_taken / player_data.max_hp

    # 玩家表现好 → 略微增加难度
    if hp_ratio > 0.8 and clear_speed < 0.7:
        performance_score = max(0.5, performance_score - 0.05)
    # 玩家表现差 → 略微降低难度
    elif hp_ratio < 0.3 or clear_speed > 1.5:
        performance_score = min(1.5, performance_score + 0.05)

func get_adjusted_enemy_multiplier(base_mult: float) -> float:
    return base_mult * performance_score

func get_adjusted_drop_rate(base_rate: float) -> float:
    # 表现差时略微提高掉落率
    return base_rate * (2.0 - performance_score)
```
