## Godot MCP 辅助核验方式

Stage 7 的体验复核优先使用 Godot MCP 辅助执行和留证，但必须区分编辑器侧核验与真实导出包 smoke。

- MCP 前置条件：
  - Godot editor 已打开当前项目。
  - Godot MCP 插件已启用。
  - `get_bridge_status` 返回 bridge 正常，且 `isGodotConnected=true`。
- 推荐 MCP 动作：
  - `open_scene`：定位主菜单、营地、结算页、设置页、挑战层入口等场景。
  - `play_scene`：执行主菜单、新局、FS/CL 入口、设置、结算返回等运行流。
  - `find_signal_connections` / `analyze_signal_flow`：辅助核验 UI 按钮、设置项、开始游戏、结算返回等交互链路。
- 证据要求：
  - 每条复核记录至少包含执行人、日期、平台、Godot 版本、构建 SHA、MCP 状态、操作路径、结果、截图/录像或日志。
  - 证据文件统一归档到 `test/manual_review_evidence/<执行日期>/`，后续已开发模块人工复核继续沿用该目录结构。
  - `MR-EXPORT-001` 必须使用真实导出包启动，MCP 只能作为编辑器侧辅助，不得替代发布包 smoke。
