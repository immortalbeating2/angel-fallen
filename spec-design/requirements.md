# 需求规格说明书

## 1. 功能需求

### 1.1 玩家系统

#### FR-P01: 玩家移动
- 玩家通过 WASD / 方向键控制角色八方向移动
- 移动速度受属性和Buff影响
- 支持冲刺（Shift），消耗耐力值，冷却恢复

#### FR-P02: 玩家属性
| 属性 | 说明 | 基础值 |
|------|------|--------|
| HP | 生命值 | 100 |
| MaxHP | 最大生命值 | 100 |
| MoveSpeed | 移动速度 | 200 px/s |
| Armor | 护甲（减伤百分比） | 0% |
| PickupRange | 拾取范围 | 64 px |
| Luck | 幸运值（影响掉落品质） | 0 |
| CritRate | 暴击率 | 5% |
| CritDamage | 暴击伤害倍率 | 150% |
| CooldownReduction | 冷却缩减 | 0% |
| XPMultiplier | 经验倍率 | 1.0 |
| Stamina | 耐力值（冲刺消耗） | 100 |

#### FR-P03: 玩家角色选择
- 至少提供 6 个可选角色
- 每个角色拥有独特的初始武器和被动技能
- 角色通过 Meta 进度解锁

#### FR-P04: 玩家死亡与复活
- HP 归零时角色死亡，本局结束（永久死亡）
- 死亡后展示本局统计数据（存活时间、击杀数、获得金币等）
- 获得的 Meta 货币保留，用于永久升级

---

### 1.2 武器与技能系统

#### FR-W01: 武器机制
- 武器自动攻击，无需玩家手动操作
- 每个武器有独立的攻击间隔、伤害、攻击范围、弹道模式
- 武器可升级（最高 Lv.8），每级提升属性或解锁新效果
- 同时最多装备 6 个武器

#### FR-W02: 武器类型（初始至少实现以下类型）
| 武器 | 弹道类型 | 说明 |
|------|----------|------|
| 魔法飞弹 | 追踪弹 | 自动追踪最近敌人 |
| 圣光环 | 环绕 | 围绕玩家旋转的光环 |
| 火焰喷射 | 锥形AOE | 朝玩家面朝方向喷射 |
| 闪电链 | 链式 | 命中后弹射至附近敌人 |
| 冰霜新星 | 圆形AOE | 以玩家为中心的冰冻爆发 |
| 飞刀 | 直线弹幕 | 高速直线投射物 |

#### FR-W03: 被动技能
- 被动技能提供永久属性加成或特殊效果
- 同时最多装备 6 个被动技能
- 被动技能可升级（最高 Lv.5）

#### FR-W04: 武器进化
- 特定武器 + 特定被动技能满级后可进化为高级武器
- 进化武器拥有全新的攻击模式和大幅增强的属性

---

### 1.3 敌人系统

#### FR-E01: 敌人生成
- 敌人从屏幕外围生成，朝玩家方向移动
- 生成频率和敌人强度随游戏时间递增
- 支持波次系统：每隔固定时间触发精英波次

#### FR-E02: 敌人类型
| 类型 | 行为 | 说明 |
|------|------|------|
| 普通 | 直线追踪 | 基础敌人，缓慢移动 |
| 快速 | 快速追踪 | 高速低血量 |
| 坦克 | 缓慢追踪 | 高血量高伤害 |
| 远程 | 保持距离+射击 | 远程攻击型 |
| 精英 | 追踪+特殊技能 | 带有特殊能力的强化敌人 |
| Boss | 复杂AI | 阶段性Boss，多种攻击模式 |

#### FR-E03: 敌人掉落
- 所有敌人死亡后掉落经验宝石
- 按概率掉落：金币、回复道具、宝箱、磁铁（吸取所有掉落物）
- 精英/Boss 掉落高品质物品和更多经验

---

### 1.4 升级与进度系统

#### FR-L01: 经验与升级
- 拾取经验宝石获得经验值
- 升级所需经验随等级递增（公式：`base_xp * level ^ 1.3`）
- 升级时暂停游戏，弹出技能选择界面

#### FR-L02: 升级选择
- 每次升级随机提供 3 个选项（受幸运值影响品质）
- 选项包括：新武器、武器升级、新被动、被动升级
- 已满级的武器/被动不再出现
- 支持重新随机（消耗金币，每局限定次数）

#### FR-L03: Meta 进度（永久升级）
- 每局游戏结束获得 Meta 货币（基于表现）
- Meta 升级树提供永久属性加成
- 解锁新角色、新武器、新被动技能
- Meta 进度存储在本地存档中

---

### 1.5 Roguelike 地图系统

#### FR-R01: 程序化地图生成
- 使用 BSP / Wave Function Collapse 算法生成随机地图
- 地图由多个房间（Room）和走廊（Corridor）组成
- 每个房间有不同的地形和敌人配置

#### FR-R02: 地图结构
- 每局游戏包含多个楼层（Floor），难度递增
- 每个楼层包含：
  - 起始房间（安全区）
  - 普通战斗房间（3-5个）
  - 精英房间（1-2个）
  - 商店房间（1个）
  - 宝藏房间（0-1个，随机出现）
  - Boss 房间（1个，通往下一层的入口）

#### FR-R03: 房间事件
- 进入房间触发战斗，清除所有敌人后解锁出口
- 商店房间：使用金币购买武器/被动/回复道具
- 宝藏房间：解谜或挑战后获得稀有奖励
- 随机事件房间：触发随机事件（祝福/诅咒/NPC交互）

#### FR-R04: 小地图
- HUD 显示当前楼层的小地图
- 已探索房间显示图标（战斗/商店/Boss等）
- 未探索房间显示为灰色问号

---

### 1.6 UI 系统

#### FR-U01: 主菜单
- 开始游戏（角色选择 → 进入游戏）
- Meta 升级（永久升级树）
- 图鉴（已解锁的武器/被动/敌人/角色）
- 设置（音量、画面、按键绑定）
- 退出

#### FR-U02: 游戏内 HUD
- 生命值条、耐力条
- 经验值条 + 当前等级
- 已装备武器图标（显示等级）
- 已装备被动图标（显示等级）
- 游戏计时器
- 击杀计数
- 金币数量
- 小地图

#### FR-U03: 升级选择界面
- 暂停游戏
- 展示 3 个选项卡片（图标 + 名称 + 描述 + 等级）
- 支持键盘/鼠标选择
- 重新随机按钮（显示剩余次数）

#### FR-U04: 暂停菜单
- ESC 暂停游戏
- 显示当前属性面板
- 继续 / 设置 / 放弃本局

#### FR-U05: 结算界面
- 存活时间、击杀数、获得经验、获得金币
- 获得的 Meta 货币
- 解锁成就提示
- 返回主菜单

## 2. 非功能需求

### 2.1 性能要求

| 指标 | 目标值 |
|------|--------|
| 帧率 | 稳定 60 FPS |
| 同屏敌人数 | ≥ 500 |
| 同屏弹幕数 | ≥ 200 |
| 内存占用 | ≤ 512 MB |
| 启动时间 | ≤ 3 秒 |
| 存档加载 | ≤ 1 秒 |

### 2.2 可维护性
- 武器/敌人/被动技能通过 Resource 配置文件定义，支持热加载
- 数值平衡通过外部 JSON/CSV 配置表驱动
- 模块间低耦合，使用信号（Signal）通信

### 2.3 可扩展性
- 支持通过 Mod 系统添加新武器/敌人/角色
- 地图生成算法可插拔替换
- 预留多人联机接口（未来版本）

### 2.4 存档系统
- 自动保存 Meta 进度
- 单局游戏不支持中途存档（Roguelike 设计）
- 存档数据加密防篡改

### 2.5 本地化
- 支持中文（简体）和英文
- 所有文本通过 i18n 键值对管理
- UI 布局适配不同语言文本长度

### 2.6 无障碍
- 支持完全键盘操作
- 支持手柄输入
- 色盲模式（可选）
- UI 字体大小可调
