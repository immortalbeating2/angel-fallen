# 实施路线图

## 总体规划

**MVP 版本**：20-24周（单人开发）  
**完整版本**：后续 v1.1 / v1.2 / v1.3 增量发布

MVP 范围：2章（F1-F6）+ 2 Boss + 基础叙事 + 4种武器 + AI生成素材

```
Phase 0 ── Phase 1 ── Phase 2 ── Phase 3 ── Phase 4 ── Phase 5 ── Phase 6
基础搭建    核心战斗    Roguelike   叙事与环境   内容填充    打磨优化    发布准备
(2周)       (4周)       (4周)       (3周)        (4周)       (3周)       (2周)
```

## 风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 500敌人性能不达标 | 中 | 高 | Phase 1 提前做性能原型验证；备选：降至300 + MultiMesh视觉欺骗 |
| BSP地图生成质量差 | 中 | 中 | 准备WFC备选算法；可退化为预制房间随机组合 |
| 数值平衡失控 | 高 | 中 | 自动化模拟测试；配置表外置便于快速调整；CDR/护甲已设上限 |
| AI素材质量不达标 | 高 | 中 | 多方案并行 (SD + MJ + 手绘修正)；像素风容错率高；预留手动优化时间 |
| AI素材风格不一致 | 中 | 中 | 固定种子/LoRA/提示词模板；Aseprite统一像素化后处理 |
| 单人开发进度延迟 | 高 | 中 | MVP只做2章；P2任务可砍；后续版本渐进发布 |
| 叙事系统复杂度超预期 | 中 | 低 | MVP只实现线性叙事；分支选择简化为2选1 |
