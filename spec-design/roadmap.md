# 实施路线图

## 总体规划

总工期预估：16-20周（单人/小团队）

```
Phase 0 ──── Phase 1 ──── Phase 2 ──── Phase 3 ──── Phase 4 ──── Phase 5
基础搭建      核心战斗      Roguelike     内容填充      打磨优化      发布准备
(2周)         (4周)         (4周)         (3周)         (3周)         (2周)
```

## 风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 500敌人性能不达标 | 中 | 高 | 提前Phase1做性能原型验证；备选方案：降低到300+视觉欺骗 |
| BSP地图生成质量差 | 中 | 中 | 准备WFC备选算法；可退化为预制房间随机组合 |
| 数值平衡失控 | 高 | 中 | 自动化模拟测试；配置表外置便于快速调整 |
| 美术资源不足 | 高 | 低 | 先用占位符开发；可使用像素风降低美术门槛 |
| 单人开发进度延迟 | 高 | 中 | 优先核心玩法，砍掉P2任务；Phase3内容可缩减 |

## 关键决策点

| 时间点 | 决策 | 依据 |
|--------|------|------|
| Phase 1 结束 | 核心战斗是否有趣？ | 内部试玩反馈 |
| Phase 2 中期 | BSP vs WFC 地图算法选择 | 生成质量和性能 |
| Phase 3 结束 | 是否需要缩减内容范围？ | 进度评估 |
| Phase 4 中期 | 是否需要额外优化周期？ | 性能测试结果 |
| Phase 5 开始 | 目标发布平台确认 | 兼容性测试结果 |
